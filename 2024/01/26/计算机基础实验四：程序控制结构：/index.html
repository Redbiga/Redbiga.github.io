#include “stdio.h”

#include “string.h”

void outputs(char *str)

{

char buffer[16]; //系统对bufff的写入str字符串没有进行越界检查，可能会覆盖返回地址

strcpy(buffer,str);

printf(“%s\n”,buffer);
}

void hacker(void)

{

printf(“being hacked\n”);

}

int main(int argc,char *argv[])

{

outputs(argv[1]);

printf(“yes\n”);

return 0;

}

编译运行程序：

系统为了防范缓冲区溢出漏洞，设置了栈地址空间的随机分配。
要实现上面的漏洞，要先关闭系统栈的随机分配
（这条命令关闭系统栈的随机分配）
（-fno-stack-protector -z execstack 多加的选项是目的是）

修改优化更加隐蔽的攻击：

原理：上面会报段错误和无法打印yes的原因是hanker过程的ret指令的返回地址是错误的。当执行hanker过程的ret指令的时候，是把此时的栈顶元素当中返回地址送到esp中，而上面的程序在执行到ret指令时此时的main栈帧的栈低元素当中了返回地址，但是这个返回地址是错误的，或者说是非法的（此时可能是把非静态变量或者是保存的寄存器的值当作了返回地址）。只有在复制bufff数组的时候把原本main栈帧中的返回地址+4的内存中写入正确的返回地址，也即是outputs的返回地址：0x8049229
